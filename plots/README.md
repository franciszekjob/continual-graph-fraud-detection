# Result figures

Generated by `scripts/plot_results.py` from the pulled ARES JSON outputs
(`*/outputs/continual_*.json`, `*/outputs/static_*.json`). Regenerate with:

```bash
uv run python scripts/plot_results.py
```

## Figures

| file | what it shows |
|------|----------------|
| `summary_heatmap.png` | ContinualAverage ROC-AUC, model × strategy — the one-glance overview |
| `heatmaps_grid.png` | all per-run 6×6 (learned × evaluated) matrices in one figure |
| `heatmap_<run>.png` | one per run, full 6×6 ROC-AUC matrix |
| `continual_average.png` | continual average vs. static baseline per run (random = 0.5 line) |
| `transfer_metrics.png` | Forward / Backward Transfer per run |
| `training_times.png` | total training cost per run (log scale, coloured by device) |
| `training_time_per_concept.png` | per-concept training time across the stream |

## Headline results

- **OCGNN / Replay** (0.69) and **DOMINANT / Replay with V-features** (0.73) are
  the only configs meaningfully above random. Replay clearly helps over Naive.
- **V-features matter a lot for DOMINANT**: 52 features → 0.51, 391 features → 0.73.
- **DOMINANT is the compute beast**: 7.6 h to train the V-feature run over the 6
  concepts vs. seconds–minutes for OCGNN/CoLA. This is why DOMINANT/Naive and
  /Cumulative OOM-killed at full scale.
- Backward Transfer is ~0 everywhere (little catastrophic forgetting in this
  setup); Forward Transfer tracks the overall ROC-AUC level.

## Two data caveats (important)

1. **The CPU `continual_dominant_replay.json` is actually the V-features run.**
   `JsonOutputWriter` names every replay run `continual_dominant_replay.json`
   regardless of `--use-v-features`, so the 391-feature run overwrote that name.
   Its ContinualAverage (0.7288350…) is byte-identical to the V-features Slurm
   log, so it is labelled **DOMINANT / Replay (V-feat)** here. The GPU file is
   the 52-feature run.

2. **`dominant_naive` is excluded.** The only `dominant_naive` JSON reports
   ContinualAverage 0.599 with 7.6 s total training — implausible at full scale
   (the real DOMINANT replay took hours) and every `dominant_naive` Slurm job
   OOM-killed. It is an old capped smoke test and would contradict the real
   "DOMINANT/Naive ran out of memory" story, so it is left out.
